# ZNXP 终极选品系统蓝图

生成日期：2026-05-12

## 一句话定位

ZNXP 要做的不是“商品列表工具”，而是跨境电商的商品机会决策操作系统：

从全网信号发现机会，自动判断能不能赚钱、能不能供应、能不能差异化、能不能投放、适不适合当前店铺，并把结果推进到上架、测试、放量和复盘。

## 终局标准

市面上最强的选品系统必须同时做到 10 件事：

1. 数据比别人全：商品、广告、搜索、社媒、竞品店铺、供应链、订单、利润都进入同一张机会图谱。
2. 判断比别人准：不是只看热度，而是综合需求、增长、竞争、利润、供应链、内容、店铺匹配和风险。
3. 解释比别人清楚：每一个推荐都能说明证据、反证、风险和下一步。
4. 动作比别人快：发现机会后自动查竞品、找供应商、算利润、生成改款、做素材、建上架草稿。
5. 学习比别人强：上架后的销售、广告、退款、评论、库存会反哺评分模型。
6. 更懂每个店：同一个商品，对不同店铺给不同结论。
7. 更懂内容电商：把 TikTok、小红书、Meta 广告、UGC、达人表现纳入商品判断。
8. 更懂供应链：能判断这款货是否可拿、可改、可控成本、可稳定交付。
9. 更懂风险：提前识别侵权、禁售、物流、季节、质量、同质化和价格战风险。
10. 更像团队大脑：支持多人协作、审批、Pipeline、报告、周会和经营复盘。

## 产品北极星指标

产品不要只看“推荐了多少商品”，应该看用户是否真的赚到钱。

| 指标 | 定义 | 目标 |
| --- | --- | --- |
| Opportunity-to-Test Rate | 被系统推荐后进入测试的比例 | 30%+ |
| Test-to-Winner Rate | 测试商品变成可放量商品的比例 | 10%-20% |
| Time to First Launch | 从发现到上架测试的时间 | 小于 24 小时 |
| Decision Confidence | 每个推荐的证据完整度 | 80 分以上 |
| Gross Margin Accuracy | 预测利润和真实利润偏差 | 小于 15% |
| Store Fit Accuracy | 系统判断适合店铺的商品最终表现 | 明显高于随机选品 |
| Learning Loop Coverage | 有销售/广告/退款回流的商品占比 | 70%+ |

## 核心护城河

### 1. Product Opportunity Graph

把全网所有商品信号归一成“商品机会图谱”。

实体：

- 商品实体：统一后的商品机会，不等于单个平台商品。
- 来源商品：Amazon/Etsy/Shopify/TikTok/小红书/Meta/GMC/供应商链接里的具体商品。
- 竞品店铺：卖这个品或类似品的店铺。
- 广告素材：正在跑或历史跑过的广告。
- 社媒内容：视频、笔记、达人内容、评论。
- 搜索需求：Google/GMC/Amazon 搜索词和趋势。
- 供应商报价：成本、MOQ、交期、评分、可定制项。
- 店铺表现：上架后的销售、转化、退款、利润、广告表现。

关键能力：

- 同款归并：标题相似、图片相似、语义相似、价格/类目相似、URL 规则。
- 信号时间线：每天记录热度、广告、价格、评论、销量、库存、排名变化。
- 证据链：每个结论能追溯到来源 URL、抓取时间和置信度。

### 2. Opportunity Score 3.0

总分必须拆解，不做黑盒。

```text
Ultimate Opportunity Score =
  demand_score      * 0.18
+ growth_score      * 0.14
+ competition_score * 0.12
+ profit_score      * 0.16
+ supply_score      * 0.12
+ creative_score    * 0.10
+ store_fit_score   * 0.10
+ launch_score      * 0.08
- risk_penalty
```

维度解释：

| 分数 | 判断什么 | 核心信号 |
| --- | --- | --- |
| demand_score | 是否有真实需求 | 搜索量、评论数、收藏、销量、订单、GMV、TikTok/小红书互动 |
| growth_score | 是否正在变热 | 7/14/30 天增长、广告新增、视频新增、竞品上新 |
| competition_score | 是否还有机会进入 | 同款密度、广告主数量、价格拥挤度、头部集中度、差异化空间 |
| profit_score | 是否能赚钱 | 成本、售价、运费、平台费、广告费、退款率、库存成本 |
| supply_score | 是否能稳定供货 | 供应商数量、MOQ、交期、评价、样品状态、可定制程度 |
| creative_score | 是否适合内容传播 | 视觉冲击、演示性、情绪价值、UGC 难度、素材复用空间 |
| store_fit_score | 是否适合当前店铺 | 店铺品类、客单价、受众、历史赢家、品牌调性 |
| launch_score | 是否适合立刻测试 | 素材是否齐、供应商是否定、页面是否能快建、风险是否可控 |
| risk_penalty | 是否有硬风险 | 侵权、禁售、物流、季节、质量、政策、价格战 |

每个商品必须输出：

- 总分
- 各维度分
- 关键证据
- 反面证据
- 风险等级
- 建议动作：观察、验证、找样、上架、测试、放量、淘汰

### 3. Store Fit Engine

最强系统必须回答“这款适不适合我的店”，而不是只说“这款很热”。

店铺画像字段：

- 主营类目
- 客单价区间
- 历史销售商品
- 历史失败商品
- 热卖商品共同特征
- 用户画像
- 可接受毛利率
- 可接受发货时效
- 禁选品类
- 品牌风格
- 内容渠道偏好

输出：

- 适配度分数
- 为什么适合
- 为什么不适合
- 需要怎么改款才适合
- 建议售价和上架形式
- 推荐素材角度

### 4. Closed-loop Learning

上架后必须回流真实结果。

回流数据：

- Shopify 订单
- 退款
- 转化率
- 加购率
- 客单价
- 毛利
- 广告花费
- ROAS
- 评论
- 客服问题
- 库存周转

回流后自动更新：

- 商品评分权重
- 店铺画像
- 类目机会判断
- 供应商可信度
- 素材风格表现
- 风险模型

没有这个闭环，选品系统会停留在“看起来很聪明”，不会真正变准。

## 功能模块终局版

### A. 机会雷达

入口：`/opportunities`

目标：每天告诉用户最该看的商品机会。

能力：

- 今日新增机会
- 7 天快速上涨
- 低竞争高利润
- 适合我店铺
- 竞品突然加码
- 广告素材突然增多
- 供应链成本下降
- 小红书/TikTok 内容爆发
- 季节性提前预警
- 风险机会分层

关键视图：

- 机会卡片
- 热力地图
- 类目趋势
- 平台趋势
- 信号来源分布
- 一键生成选品报告
- 一键加入 Pipeline

### B. 商品情报详情页

入口：`/intelligence/products/:id`

目标：一个页面完成选品决策。

模块：

- 商品摘要：一句话机会判断
- 评分雷达：需求、增长、竞争、利润、供应、内容、店铺适配、风险
- 趋势时间线：搜索、社媒、广告、价格、评论、销量
- 竞品矩阵：谁在卖、卖多少钱、怎么展示、广告跑多久
- 素材拆解：广告 hook、视频结构、评论痛点、视觉方向
- 利润模型：成本、售价、费用、保本 ROAS、敏感性分析
- 供应商匹配：报价、MOQ、交期、可定制、样品状态
- 风险雷达：侵权、禁售、物流、质量、季节、价格战
- 改款建议：差异化方向、组合套装、个性化字段、包装方案
- Launch Plan：测试预算、素材数量、渠道、KPI、止损线
- AI 报告：可导出 PDF/Markdown

### C. 竞品情报中心

入口：`/competitors`

目标：知道竞品在做什么，并转化成自己的行动。

能力：

- 监控 Shopify 店铺
- 监控商品上新
- 监控价格变化
- 监控折扣和库存信号
- 监控广告素材变化
- 监控落地页结构
- 监控热销品类变化
- 识别竞品正在放量的商品
- 对比我的店铺和竞品差距

高级能力：

- 竞品得分
- 竞品商品克隆风险
- 竞品新爆品预警
- 竞品广告素材复盘
- 竞品周报

### D. 供应链验证中心

入口：`/sourcing`

目标：把“看起来能卖”变成“真的能供”。

能力：

- 图片搜同款供应商
- 关键词搜供应商
- 1688/AliExpress/Alibaba/本地供应商录入
- MOQ、交期、报价、样品、质检记录
- 可定制项管理
- 包装和品牌化能力
- 供应商评分
- 风险记录
- 供应商对比
- 样品流程

输出：

- 推荐供应商
- 推荐采购价
- 建议售价
- 可改款方向
- 打样清单
- 风险提示

### E. 利润实验室

入口：`/profit-lab`

目标：让用户知道这个品到底能不能赚钱。

能力：

- 成本模板
- 平台费模板
- 运费模板
- 支付费
- 广告 CPA/ROAS 假设
- 退款率
- 汇率
- 税费
- 仓储费
- 折扣策略
- 多价格带模拟
- 敏感性分析

输出：

- 毛利率
- 贡献利润
- 保本 ROAS
- 保本 CPA
- 建议售价
- 可接受广告成本
- 放量阈值

### F. Creative Intelligence

入口：`/creative-intelligence`

目标：把内容表现纳入选品判断。

能力：

- 广告素材库
- TikTok/小红书视频库
- Hook 提取
- 卖点提取
- CTA 提取
- 视频结构拆解
- 评论痛点聚类
- 素材版本管理
- 图片/视频生成
- UGC brief 生成
- 渠道尺寸适配

输出：

- 5 条广告 hook
- 3 套视频脚本
- 3 个落地页首屏方案
- 10 条评论区话术
- 3 个 UGC 达人 brief

### G. Launch 工作台

入口：`/launches`

目标：把选品推进到测试和放量。

状态流：

```text
Discovered -> Validating -> Sourcing -> Sampled -> Listing Draft
-> Launched -> Testing -> Scaling -> Winner / Killed / Paused
```

能力：

- 测试计划
- 上架任务
- 素材任务
- 预算计划
- KPI 设置
- 止损线
- 投放结果录入/API
- 自动复盘
- 下一步建议

核心判断：

- 继续测试
- 换素材
- 降价/提价
- 改款
- 放量
- 淘汰

### H. AI Strategy Copilot

入口：全局浮层 + 每个详情页

目标：让用户用自然语言问整个系统。

示例问题：

- “今天有哪些适合我的刺绣礼品店的新机会？”
- “这款为什么推荐？”
- “这款和我历史爆品像不像？”
- “如果采购价 6 美元，卖 29.99 美元，最多能接受多少 CPA？”
- “帮我找 10 个竞品正在放量的宠物刺绣商品。”
- “给这款生成 14 天测试计划。”
- “过去 30 天失败商品有什么共同原因？”

Copilot 不能只聊天，必须能执行动作：

- 加入 Pipeline
- 生成报告
- 创建 Launch
- 生成素材任务
- 查询供应商
- 创建上架草稿
- 标记风险
- 分配负责人

## AI Agent 架构

### Agent 类型

| Agent | 输入 | 输出 |
| --- | --- | --- |
| Discovery Agent | 数据源、类目、店铺画像 | 候选机会 |
| Scoring Agent | 商品实体和信号 | 可解释评分 |
| Competitor Agent | 商品/关键词/店铺 | 竞品矩阵和变化 |
| Sourcing Agent | 商品图/关键词/成本要求 | 供应商和报价 |
| Profit Agent | 成本、售价、广告假设 | 利润模型和价格建议 |
| Creative Agent | 商品、竞品广告、评论 | Hook、脚本、素材 brief |
| Risk Agent | 商品、图片、标题、类目 | 风险报告 |
| Launch Agent | 商品、店铺、素材、成本 | 测试计划 |
| Review Agent | 销售、广告、退款、评论 | 复盘和下一步 |

### 工作流模板

```text
发现机会
-> 合并同款
-> 采集信号
-> 评分
-> 查竞品
-> 算利润
-> 找供应商
-> 查风险
-> 生成改款建议
-> 生成素材 brief
-> 建 Launch Plan
-> 人工确认
-> 上架草稿
-> 测试
-> 复盘
```

### Agent 运行要求

- 每一步都要有输入快照。
- 每一步都要有输出结构化 JSON。
- 每一步都能重试。
- 每一步都能人工覆盖。
- 每一步都记录模型、成本、耗时。
- 关键动作必须有审批。

建议主导航改成 7 个系统：

1. 指挥舱：Dashboard、机会雷达、周报。
2. 选品系统：机会库、商品情报、商品 Pipeline。
3. 竞品系统：店铺监控、广告监控、竞品报告。
4. 供应链系统：供应商、报价、样品、利润实验室。
5. 内容系统：素材情报、AI 图片、AI 视频、UGC brief。
6. Launch 系统：上架、测试、投放、复盘。
7. 设置系统：店铺、数据源、团队、权限、成本。

## 页面优先级

| 优先级 | 页面 | 为什么先做 |
| --- | --- | --- |
| P0 | 商品情报详情页 | 决策核心页面 |
| P0 | 机会雷达 | 替代普通推荐列表 |
| P0 | 利润实验室 | 直接决定是否赚钱 |
| P0 | 供应商匹配 | 从看品进入可执行 |
| P1 | 竞品监控 | 建立差异化情报 |
| P1 | Launch 工作台 | 形成测试闭环 |
| P1 | AI 报告中心 | 形成高感知价值 |
| P2 | Copilot | 提升效率和粘性 |
| P2 | 团队 Pipeline | 服务团队用户 |

## 商业化版本设计

| 版本 | 目标用户 | 核心限制 |
| --- | --- | --- |
| Starter | 单店小卖家 | 每日机会数、数据源、报告数有限 |
| Pro | 成熟 Shopify 卖家 | 开放评分、竞品、供应商、Launch |
| Team | 选品/运营团队 | 多成员、审批、Pipeline、周报 |
| Agency | 代运营/服务商 | 多店铺、多客户、白标报告 |
| Enterprise | 品牌/工厂/大卖 | API、私有数据源、自定义模型、权限审计 |

收费点：

- 数据源额度
- 竞品监控店铺数
- AI 报告数
- 视频/图片生成额度
- Launch 项目数
- 团队席位
- API 调用
- 高级数据回流

## 极致体验标准

### 商品卡片

普通商品卡片不够。极致商品卡片应展示：

- 机会分
- 趋势箭头
- 利润区间
- 竞争强度
- 供应链状态
- 店铺适配度
- 风险等级
- 推荐动作

### 商品详情

用户打开详情页 30 秒内必须知道：

- 这个品能不能做
- 为什么
- 最大风险是什么
- 预计赚多少钱
- 谁在卖
- 找谁供货
- 怎么改款
- 怎么拍素材
- 下一步做什么

### AI 报告

报告必须像投资备忘录：

- Executive Summary
- Opportunity Thesis
- Evidence
- Market Signals
- Competitor Landscape
- Profit Model
- Supply Chain
- Creative Strategy
- Risk Assessment
- Launch Plan
- Go/No-Go Decision

## 实施路线图

### Phase 0：产品骨架重构，1-2 周

目标：把系统设计定下来，避免后面边做边乱。

交付：

- 新导航信息架构
- 核心表结构设计
- Opportunity Score 3.0 公式
- 商品详情页 PRD
- 机会雷达 PRD
- 数据源接入规范
- Agent 输出 JSON Schema

### Phase 1：决策底座，4-6 周

目标：让每个商品可解释、可算钱、可判断风险。

交付：

- `product_entities`
- `product_sources`
- `product_signals`
- `opportunity_scores`
- 商品情报详情页
- 利润实验室
- 风险雷达
- Shopify 销售回流基础版

验收：

- 每个推荐商品有 8 维评分。
- 每个分数有证据。
- 每个商品能算利润。
- 每个商品有 Go/No-Go 建议。

### Phase 2：竞品和供应链，6-8 周

目标：判断机会是否可进入、能否供货、如何差异化。

交付：

- 竞品店铺监控
- 竞品商品事件
- 供应商匹配
- 样品流程
- 改款建议
- 创意拆解

验收：

- 能监控竞品上新/改价/广告变化。
- 能为商品找到候选供应商。
- 能输出改款和素材方向。

### Phase 3：Launch 闭环，6-8 周

目标：从选品到测试结果闭环。

交付：

- Launch 工作台
- 测试计划
- 素材任务
- 广告表现录入/API
- 复盘报告
- 模型回流

验收：

- 每个 Launch 有预算、KPI、止损线。
- 测试结果能影响后续推荐。
- 系统能给出放量/淘汰建议。

### Phase 4：Agent 和团队化，8-12 周

目标：把系统变成团队工作流和 AI 决策大脑。

交付：

- Workflow Engine
- Copilot
- 多角色权限
- Pipeline
- 审批
- 通知
- 周报/月报
- API/Webhook

验收：

- 用户可以用自然语言完成核心操作。
- 团队可以协作推进商品。
- 管理者可以看选品转化和利润。

## 第一版必须做的 12 个任务

如果马上开干，建议按这个顺序：

1. 设计 `product_entities/product_sources/product_signals/opportunity_scores` 表。
2. 重写推荐评分为 Opportunity Score 3.0。
3. 改造商品详情页，显示评分、证据、利润、风险。
4. 增加利润模型 API 和前端利润实验室。
5. 增加 Shopify 订单/退款回流。
6. 增加商品 Pipeline 状态。
7. 增加供应商报价和样品流程。
8. 增加竞品店铺监控表和基础扫描任务。
9. 增加 AI 商品报告生成。
10. 增加 Launch Plan 数据模型和页面。
11. 增加素材资产库。
12. 增加数据源管理和数据质量面板。

这 12 个做完，产品会从“能用的选品工具”变成“真正能卖给专业卖家的选品系统”。

## 不应该优先做的事

这些事看起来炫，但早期不要优先：

- 先做复杂 BI 大屏，但没有商品级决策。
- 先做聊天机器人，但没有结构化数据支撑。
- 先接很多数据源，但不做统一商品实体。
- 先做大量 AI 文案，但不算利润和风险。
- 先做社交分享和营销页，而核心工作流还没闭环。
- 先做移动端 App，而桌面端决策效率还没做好。

## 技术架构建议

### 后端

- FastAPI 保留。
- PostgreSQL 作为主库。
- Redis 用于缓存和任务队列。
- 长任务迁移到 Arq/Celery/RQ。
- Alembic 管理迁移。
- 外部数据源统一走 `connectors` 层。
- AI 输出统一 JSON Schema。
- 所有任务有状态、日志、重试、成本。

### 前端

- Next.js 保留。
- 页面从“功能页”升级为“工作台”。
- 商品卡片组件重构为机会卡片。
- 图表统一用 Recharts 或更稳定图表层。
- 大量数据表格需要虚拟滚动和保存视图。
- 所有关键按钮都要有 loading/error/empty 状态。

### 数据层

- 原始数据不要丢。
- 清洗数据和原始数据分层。
- 评分模型要版本化。
- 人工修正要记录。
- 证据和结论分开存。

## 风险和约束

| 风险 | 应对 |
| --- | --- |
| 外部平台 API 限制 | 官方 API + 合规爬取 + 手动导入三套方案 |
| 爬虫不稳定 | 队列、重试、代理池、失败告警、数据质量分 |
| AI 幻觉 | 结构化输入、证据引用、置信度、人工审批 |
| 数据太杂 | 统一信号层和商品实体归并 |
| 推荐不准 | 销售/广告/退款回流，按店铺训练偏好 |
| 用户不执行 | 每个推荐给下一步动作和 Launch Plan |
| 成本失控 | 任务成本记录、额度、缓存、模型分层 |

## 最终产品形态

用户每天打开 ZNXP，看到的不是一堆商品，而是：

- 今天最值得做的 10 个机会。
- 哪些机会适合我的店。
- 每个机会为什么值得做。
- 每个机会预计能赚多少钱。
- 谁在卖、谁在投、怎么投。
- 供应商在哪里，成本是多少。
- 怎么改款避免同质化。
- 素材怎么拍，广告怎么测。
- 上架后结果如何，下一步该放量还是淘汰。

这才是“市面上最强选品系统”的标准。
