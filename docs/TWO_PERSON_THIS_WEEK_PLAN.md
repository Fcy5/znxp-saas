## 三、协作边界

| 角色 | 本周负责方向 | 本周核心职责 |
| --- | --- | --- |
| 共同开发者 A | 规则与数据层 | 字段、状态、专题池、候选池、批量处理、数据视图 |
| 你 / 我方 B | 判断与体验层 | 评分、理由、看板、选品库体验、规则解释、策略读取 |

协作原则：

- A 优先稳定后端结构和接口
- B 优先稳定评分层、页面层、解释层
- 公共文件尽量减少同时改动
- 如果必须同时动同一文件，先锁字段口径，再改实现

## 八、本周偏差说明

本周没有完全按最初按日任务表逐条推进，而是按依赖关系重排：

1. 先打通候选池、评分、状态、看板主链路
2. 再补批量处理和复核
3. 再补本周视图隔离
4. 再补解释层和策略读取
5. 最后补前端卡片体验

这是合理偏差，不算跑偏。

原因：

- 先把链路跑通，才能暴露真实问题
- 否则只按草案做点状开发，无法验证整个系统是否可用

## 九、本周没有完成到极致的点

这周已经把选品系统做成第一版可运行产品，但离“极致”还差这几项：

### 1. 候选池质量仍可继续压

- `Summer`
- `Memorial Day`

仍有泛热品或弱定制商品混入。

### 2. 自动标签能力仍偏规则化

- 还没有做到 NLP/LLM 语义抽取
- 还没有准确率评估集

### 3. 权重仍未数据校准

- 当前是第一版业务权重
- 尚未结合真实转化反馈迭代

### 4. 策略还不可视化编辑

- 已有策略 API
- 已有只读策略面板
- 但还没有前端直接编辑策略的管理能力
